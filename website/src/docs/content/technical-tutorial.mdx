# Technical tutorial: make Beside your product-grade AI memory layer

Beside is a local-first context system for people who move between IDEs, chat,
email, meetings, browsers, design tools, and AI agents all day. It watches the
work surface, turns raw activity into structured memory, and exposes that memory
through interfaces your tools already understand.

The default setup is intentionally simple: install the desktop app, let Beside
capture locally, and ask Claude, Cursor, ChatGPT, or any MCP-compatible agent
questions that used to require digging through half a dozen apps. The technical
stack is also intentionally open. Every capture adapter, model adapter, hook,
exporter, and storage layer is designed to be replaced when your workflow needs
something more specific.

<h2 id="what-beside-gives-you">What Beside Gives You</h2>

Beside is not another note-taking app you have to babysit. It is the memory
infrastructure beside your computer.

- **A local activity timeline.** Screenshots, active windows, URLs, idle state,
  and app context are captured on your machine so the system can remember what
  actually happened.
- **A self-organising wiki.** Beside turns captures into Markdown pages under
  your local workspace. You can read them, edit them, grep them, version them,
  or let the indexer keep shaping them for you.
- **Agent-ready recall.** The MCP exporter makes your local memory available to
  Claude, Cursor, ChatGPT, Windsurf, and any agent that can call MCP tools.
- **A composable product surface.** Desktop gives you a polished always-on Mac
  experience. CLI gives developers automation and scripts. Plugins give teams a
  way to swap capabilities without forking the product.

The unique value is that Beside keeps context close to where it was created. The
AI does not start from an empty prompt, and you do not have to paste yesterday's
decisions into every new chat.

<h2 id="architecture-at-a-glance">Architecture At A Glance</h2>

Beside is built as a pipeline. Each stage has a clear job, a stable interface,
and a reason to exist independently.

```text
Desktop / CLI
    |
    v
Capture plugins  ->  Raw events  ->  Local storage
    |                                  |
    v                                  v
Model adapters  ->  Index plugins  ->  Markdown wiki
                                      |
                                      v
Hooks + exporters  ->  MCP, CLI, desktop, automations
```

The product advantage of this shape is control. You can start with the packaged
desktop app, then replace only the layer that matters: use a different model,
exclude sensitive apps, add a team-specific hook, or expose memory through a new
exporter. Beside stays one system instead of becoming a pile of one-off scripts.

<h3>Core packages</h3>

- `@beside/core` defines shared types, plugin contracts, schedules, and config
  primitives. It is the stability layer for the rest of the product.
- `@beside/runtime` runs the pipeline: capture, store, index, hook, and export.
  It is where Beside becomes an always-on memory engine.
- `@beside/cli` gives developers a scriptable control plane for starting,
  indexing, reindexing, and automation.
- `@beside/desktop` wraps the runtime in an Electron app for a native Mac
  experience with packaging, updates, permissions, and user-facing controls.

<h2 id="choose-the-right-layer">Choose The Right Layer</h2>

Beside gives you several ways to customize the same memory pipeline. Pick the
lowest layer that solves your problem.

<h3>Plugins: replace capability</h3>

Plugins are for durable product behavior. Use a plugin when you want Beside to
capture from a new source, store data differently, index with a new strategy,
call a different model, or export memory to another tool.

Current plugin layers include:

- `capture`: turns apps, screens, URLs, audio, and focus changes into raw events.
  The `native` capture plugin uses a platform helper for richer Mac capture; the
  `node` capture plugin is a portable TypeScript implementation.
- `storage`: persists events and indexes locally. This is where Beside's
  local-first promise becomes concrete.
- `model`: adapts LLMs and embedding providers. Use `ollama` for private local
  inference or `openai` when a hosted model is the right tradeoff.
- `index`: turns raw signals into useful pages, topics, timelines, and recall
  surfaces.
- `export`: publishes memory. The `mcp` exporter is the flagship path because it
  makes Beside immediately useful inside modern AI agents.

That plugin boundary is one of Beside's biggest selling points: you can tailor
the intelligence without replacing the product.

<h3>Hooks: react to moments</h3>

Hooks are for event-driven behavior. Use a hook when you want Beside to notice
something specific and take action.

Examples:

- Detect open follow-ups from Slack, Teams, Gmail, Outlook, Apple Mail, and
  meeting transcripts.
- Turn a recurring meeting into a living project page.
- Raise a reminder when a pricing thread, contract review, or launch blocker
  shows up across multiple apps.

Hooks make Beside feel proactive. They are the difference between a passive
archive and a memory layer that catches the important thread before it slips.

<h3>CLI: automate the pipeline</h3>

The CLI is the developer control surface. Use it when you want repeatable local
commands, CI-style checks, scheduled jobs, or manual reindexing.

```bash
pnpm cli start
pnpm cli index --once
pnpm cli index --reorganise
pnpm cli index --full-reindex
```

The product win: every team can keep Beside in the flow they already use. Run it
from a terminal, pair it with launchd, trigger it from a script, or keep it
behind the desktop app.

<h3>Electron desktop: ship the experience</h3>

The Electron app is the consumer-grade shell around the same runtime. It handles
packaging, permissions, native capture resources, local controls, and the
low-friction "install once and forget it" experience.

Use Desktop when you want Beside to behave like a Mac product. Use CLI when you
want automation. Use plugins and hooks when you want to change what the product
knows how to do.

<h2 id="customize-the-pipeline">Customize The Pipeline</h2>

Start with the defaults, then tune the smallest surface area needed.

<h3>1. Tune capture</h3>

Capture decides what Beside is allowed to remember. That makes it both a power
feature and a trust feature.

Common settings include:

```json
{
  "poll_interval_ms": 3000,
  "screenshot_format": "webp",
  "screenshot_quality": 45,
  "screenshot_max_dim": 1100,
  "excluded_apps": ["1Password", "Keychain Access"],
  "excluded_url_patterns": ["bank", "admin"],
  "capture_mode": "active"
}
```

Marketing translation: Beside can be always-on without being reckless. You get
ambient memory with explicit controls for sensitive apps, URLs, quality, and
overhead.

<h3>2. Pick your model posture</h3>

Use `ollama` when privacy and offline operation matter most. Use `openai` or an
OpenAI-compatible endpoint when you want hosted model quality or centrally
managed infrastructure.

```json
{
  "model": {
    "plugin": "ollama",
    "config": {
      "host": "http://localhost:11434",
      "model": "gemma4:e4b"
    }
  }
}
```

This is where Beside avoids lock-in. The memory layer stays yours even when the
model changes.

<h3>3. Add hooks for your domain</h3>

A founder might use hooks for investor follow-ups. A product team might use them
for launch risks. A support team might use them for account escalations.

```ts
export async function onIndexedPage(page, context) {
  if (page.tags.includes("pricing") && page.text.includes("follow up")) {
    await context.surface({
      title: "Pricing follow-up detected",
      body: "Beside found a pricing thread that likely needs a response."
    });
  }
}
```

The best hooks feel like judgment. They turn scattered signals into a timely
nudge.

<h3>4. Expose recall through MCP</h3>

MCP is how Beside becomes useful everywhere instead of trapped in its own UI.
Configure your agent to call Beside locally, then ask questions like:

- "What did we decide about pricing last week?"
- "Summarize the customer concerns from my last three Acme interactions."
- "What open follow-ups are hiding in Slack and email?"
- "Use my local context before drafting this reply."

The selling point is simple: your AI stops guessing from the current chat and
starts answering from your actual work history.

<h2 id="tutorial-build-your-first-workflow">Tutorial: Build Your First Workflow</h2>

This walkthrough creates a practical workflow for a product lead who wants
Beside to remember launch decisions, surface follow-ups, and make that context
available in Cursor.

<h3>Step 1: Install and run Beside</h3>

Install the desktop app for the easiest path. It gives you native packaging,
permissions, and background runtime management.

Developers can also run from source:

```bash
pnpm install
pnpm build
pnpm desktop
```

Once running, Beside starts building local memory from the apps you allow it to
observe.

<h3>Step 2: Keep capture focused</h3>

For a launch workflow, you might want product, support, design, and engineering
signals, but not password managers or banking pages. Add exclusions first. That
keeps trust high and noise low.

```json
{
  "capture": {
    "plugin": "native",
    "config": {
      "capture_mode": "active",
      "excluded_apps": ["1Password", "Keychain Access"],
      "excluded_url_patterns": ["billing", "bank", "admin"]
    }
  }
}
```

Beside's advantage is not "capture everything at any cost." It is durable
context with sensible boundaries.

<h3>Step 3: Let the indexer build the wiki</h3>

Run a one-off index while testing, then let the runtime continue in the
background.

```bash
pnpm cli index --once
pnpm cli index --reorganise
```

You should see Beside turn raw activity into pages, topics, and timelines. The
Markdown output is intentionally boring in the best way: it is portable,
diffable, and easy to fix by hand.

<h3>Step 4: Connect Cursor or another MCP agent</h3>

Use the MCP exporter when you want an AI agent to recall Beside memory. The MCP
server runs locally by default and can expose wiki excerpts to compatible tools.

```json
{
  "export": {
    "plugin": "mcp",
    "config": {
      "host": "127.0.0.1",
      "port": 3456,
      "transport": "http",
      "text_excerpt_chars": 5000
    }
  }
}
```

Now ask Cursor: "Before editing the launch plan, check Beside for recent launch
decisions and open follow-ups."

That is the core product promise in one sentence: the agent works with the
context that was already beside you.

<h3>Step 5: Add one hook that creates leverage</h3>

Start with follow-ups. They are easy to validate and immediately valuable.

```json
{
  "hooks": [
    {
      "plugin": "followups",
      "enabled": true
    }
  ]
}
```

Once this is working, extend the idea for your own domain: contract risk,
customer objections, roadmap churn, launch blockers, hiring loops, or anything
else that currently lives across too many tabs.

<h2 id="where-to-go-next">Where To Go Next</h2>

If you want the fastest path, use the desktop app and MCP. If you want more
control, use the CLI and configure capture, model, hooks, and exports directly.
If you want to build on Beside, write a plugin at the layer where your workflow
needs a new capability.

Beside is designed to be both a product and a platform: polished enough to use
every day, open enough to adapt to the exact way your team thinks.
